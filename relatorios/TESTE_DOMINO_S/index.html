1. Contexto Estratégico e Objetivos

▼

Com base na reunião de discovery de 25/03/2026, validamos a arquitetura híbrida para a Domino's Pizza, operando em dois segmentos distintos com necessidades diferentes:

Segmentação Dual-Track

  • B2C (Volume - Ticket Médio R$90-100): Atendimento 100% automatizado via IA conversacional. Foco em conversão rápida, baixa fricção e captura completa de dados (telefone, endereço completo, complemento).
  • B2B Emergente (Alto Valor - Ex: 70 pizzas para faculdade): Atendimento humano qualificado. Foco em construção de relacionamento, logística, faturamento e recorrência (potencial semanal/mensal).

Objetivos Principais Validados

  • Captura de Dados Perdidos: Recuperar contatos de clientes que hoje se perdem nos agregadores (iFood, 99Food).
  • Automatização Operacional: Eliminar o processo manual de lançamento de pedidos no WhatsApp, integrando diretamente ao sistema Pulse/Plenis.
  • Triagem Inteligente: Identificar e isolar leads B2B de alto valor para atendimento humano especializado, liberando gestores de pedidos B2C.
  • Expansão da Base e Recorrência: Aumentar o ticket médio via cross-sell automatizado e converter vendas B2B pontuais em contratos recorrentes.
"Nosso intuito é pegar esses pedidos que são do iFood, 99, para nossa base de dados. Quando for sair o pedido, colocar uma tag para dar prioridade para que quando ele voltar no próximo pedido, ele venha para o telefone da loja ou para IA." - Emanuel Mesquita
Estratégia focada em migrar clientes de agregadores (sem dados) para canal próprio (com dados completos).

3. Detalhamento dos Funis de Venda

▼

Funil B2C (Automatizado - Volume)

1. Entrada no WhatsApp (IA)
2. Qualificação & Coleta de Dados
3. Confirmação do Pedido + Cross-sell
4. Integração com Sistema (Pulse/Plenis)
5. Pedido Entregue
✓ Cliente na Base + Nutrição
Etapa Ação/Automação Métrica de Sucesso
1. Entrada IA inicia conversa natural. Follow-up automático se cliente para de responder (ex: após 5 min). Taxa de resposta inicial > 95%.
2. Qualificação Coleta OBRIGATÓRIA de: Nome, Telefone, Endereço Completo (complemento). Aplica "Filtro Inteligente". Taxa de Captura de Dados: 100% (vs. 0% nos agregadores).
3. Confirmação Cross-sell sistemático (bebidas, sobremesas, bordas). Confirma método de pagamento (dinheiro/cartão). Aumento do Ticket Médio (TM) via itens adicionais.
4. Integração Pedido enviado automaticamente para o sistema de produção (Pulse/Plenis). Meta: API aberta. Eliminação do lançamento manual. Redução de erro e tempo de preparo.
5. Pós-Venda Cliente entra na base para nutrição (e-mail). Tag por CEP para campanhas geolocalizadas. Recorrência de clientes capturados.

Funil B2B (Humano - Alto Valor)

1. Identificação pela IA
2. Contato Imediato do Gestor
3. Elaboração de Proposta
4. Negociação & Faturamento
5. Entrega & Satisfação
✓ Conta Corporativa (Recorrente)
Etapa Ação/Automação Métrica de Sucesso
1. Identificação IA detecta sinal B2B (quantidade, contexto, NF). NÃO entra no fluxo automatizado. Taxa de Captura B2B: >95% (identificação correta).
2. Contato Alerta em tempo real para gestor (Emanuel/Wendel). Pipeline visual separado no CRM. Tempo de primeiro contato < 1h.
3. Proposta Gestor elabora proposta comercial com condições (volume, recorrência, faturamento). Conversão de lead B2B em proposta: >70%.
5. Pós-Venda Após 3 semanas, oferta automática de "Programa de Descontos para Pedidos Recorrentes". Conversão de venda pontual em contrato recorrente.
Meta de Eficiência Financeira: Liberar +80% do tempo dos gestores (Emanuel/Wendel) da operação B2C, redirecionando para prospecção e gestão de contas B2B.
